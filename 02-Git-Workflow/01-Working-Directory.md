# Working Directory

## What is a Working Directory?

The Working Directory is the current project folder where developers create, edit, modify, and delete files. Every change made to a project starts from the Working Directory.

---

## Why Do We Use It?

The Working Directory is the place where developers perform all development activities before saving the changes in Git.

---

## Example

Suppose you create a file named `Main.java` and start writing code.

At this stage, the file exists only in the Working Directory. Git has not yet prepared or saved these changes.

---

## Key Points

- Every new file is created in the Working Directory.
- Every modification starts in the Working Directory.
- Git does not save these changes automatically.
- Changes remain in the Working Directory until they are staged.

---

## Summary

The Working Directory is the current project folder where all development begins. Every new file and every modification is first made here before moving to the next stage of the Git workflow.
