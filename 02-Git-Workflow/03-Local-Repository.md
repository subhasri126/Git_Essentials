# Local Repository

## What is a Local Repository?

The Local Repository is the place where Git permanently stores committed changes on the local machine. Every commit is saved inside the hidden `.git` directory.

---

## Why Do We Use It?

The Local Repository maintains the complete history of the project. Developers can view previous commits, restore older versions, and continue working even without an internet connection.

---

## Example

After staging the changes, execute:

```bash
git commit -m "Initial Commit"
```

Git creates a commit and stores it in the Local Repository.

---

## Key Points

- Stores committed changes.
- Maintains complete project history.
- Located inside the `.git` directory.
- Works without an internet connection.

---

## Summary

The Local Repository stores every committed version of the project and maintains the complete history on the developer's local machine.
