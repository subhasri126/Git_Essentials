# Git vs GitHub

## What is Git?

Git is a **Distributed Version Control System (DVCS)** used to track changes in a project. It helps developers maintain version history, restore previous versions, and collaborate with other developers.

---

## What is GitHub?

GitHub is a **cloud-based hosting platform** for Git repositories. It allows developers to store repositories online, collaborate with team members, and access projects from anywhere.

---

## Git vs GitHub

| Git | GitHub |
|------|---------|
| Version Control System | Cloud-based hosting platform |
| Installed on a local computer | Works online |
| Tracks project changes | Stores Git repositories |
| Works without an internet connection | Internet is required for online collaboration |
| Manages project versions | Enables collaboration and backup |

---

## Relationship Between Git and GitHub

Git and GitHub are different technologies, but they work together.

- Git manages the project locally.
- GitHub stores the project online.
- Git is used to create commits.
- GitHub is used to share and collaborate on repositories.

---

## Real-Time Example

Suppose you are developing a project on your laptop.

- You write and modify the code using Git.
- After completing your work, you push the project to GitHub.
- Your teammates can pull the latest changes from GitHub and continue development.

---

## Key Points

- Git is a software.
- GitHub is a hosting platform for Git repositories.
- Git can work without GitHub.
- GitHub uses Git to manage repositories.
- GitHub makes collaboration and backup easier.

---

## Summary

Git is used to manage project versions on a local machine, while GitHub is used to host Git repositories online. Together, they help developers track changes, collaborate efficiently, and maintain project history.
